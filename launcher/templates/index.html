Overview

This is a local command deck, not a public website.

The launcher runs only on your machine and treats each tool as a separate process. That keeps Frida, Flask dashboards, proxies, and monitors isolated from each other while still letting the launcher track status, logs, and openable interfaces.

Config Notes

How the shared path setting works

One source of truth

launcher/config.json is now the shared path source for the launcher and the imported Python helpers that use path_helpers.py.

Standalone support

If you run those scripts directly outside the launcher, they will still read the same saved config file instead of falling back to the old hardcoded developer paths.

What the test checks

The test verifies the chosen SeaMail folder, hostDB, Resource, and WeatherGet.dll so you can spot bad copies quickly.

Main

Core game and translation services

Run Game

Launches the configured Seaman executable. Uses the path set in Settings. Live process detection — status reflects whether the EXE is actually running, not just what the launcher last did.

Menu Translation Hook

Frida hook that intercepts Japanese speech balloon text and reinjects an English translation in real time. Must be running while the game is open.

Menu Replacement Overlay

Transparent tkinter overlay over the game window. Provides clickable menu shortcuts, tickle/spin interactions, and live translated speech display. Activates screensaver mode automatically on launch.

Servers

Network services the game expects to find

SMTP + POP Server

Local SMTP (port 25) and POP3 (port 110) so the game can send and receive SeaMail. Manage the inbox queue via the Mail interface tab.

SMTP + POP Debug Server

Same as above on shifted ports (SMTP 2525, POP 1100) to avoid the admin requirement for port 25 on Windows 11. Prefer this if the standard server fails to bind.

Dual HTTP + FTP Server

Local HTTP (port 80) and FTP for legacy update and file requests. Conflicts with Weather Redirection Server — only one can hold port 80.

Weather Redirection Server

Spoofs the JMA weather endpoint on port 80 so in-game weather can be controlled locally. Conflicts with Dual HTTP + FTP Server.

Tools

Live tracking and monitoring

GEM Variable Tracker Dashboard

Web dashboard at 127.0.0.1:5081 showing live game variables. Needs Creature Tracker Engine running to receive data — start either one first, they connect automatically.

Creature Tracker Engine

Memory polling engine. Discovers the creature instance by hooking the brain state machine, then reads ~295 variables directly and pushes them to the GEM dashboard.

SDP Comparison Monitor

Watches SDP script output and highlights changes between runs. Useful for diffing dialogue script behaviour.

Fix Loop / Interaction Hook v33

Frida hook for the interaction loop. Opens an interactive console — type vocab, pick <n>, spoof <word>, status, or discover. Game must be running.

3D Position Tracker Bridge

Receives UDP telemetry from the game and visualises Seaman's 3D position and state.

Live Position Tracker

Integrated Flask + Socket.IO tracker at 127.0.0.1:5025 with real-time 3D movement visualisation.

DB

Database editors and monitors

DB Manager V2

Flask editor at 127.0.0.1:5074 with change tracking and inline editing. Reads and writes the IDEA-encrypted UDB files (Seaman, User, Calender).

DB Manager V1

Original Flask editor at 127.0.0.1:5073. Simpler interface, same underlying UDB files.

DB Proxy Web Analyzer

Web dashboard at 127.0.0.1:5075 for live analysis of database calls. Conflicts with DB Proxy Monitor.

DB Proxy Monitor

UDP monitor that captures internal database traffic. Conflicts with DB Proxy Web Analyzer.

Quick DB Editor

Command-line UDB editor for direct key search and value editing without the web UI.

DNA Selector

Interactive 14×14 trait matrix at 127.0.0.1:5076. Click any cell to choose Element 1 + Element 2, see the resulting variant, and apply directly to Seaman.udb. Includes random picker.

Launcher

How the launcher itself works

task_registry.py

Single file where all tasks are registered — label, group/tab, script path, venv, and launch mode. Edit this to add, move, or reconfigure tasks.

Launch modes

detached — runs python directly, no extra console wrapper (used for overlays and GUI tools). console — opens a cmd window. background — captures stdout into the log panel.

Venvs

Two shared environments: launcher-app for Flask/web services, launcher-frida for anything using Frida or the overlay. Click Setup Env on any task to create or update its venv.

Settings

Paths stored in launcher/config.json. Project Root is the workspace folder. SeaMail Folder is where the game files live. Game Executable is the EXE to launch.
